# 02 User Journeys
*LegacyGuard Product Design — Generated 2026-03-17*

---

Here are the detailed user journey maps for LegacyGuard, designed with a strong emphasis on emotional intelligence and trust-building.

---

## FLOW 1 — New User Onboarding (15 steps)
**Emotional arc: Curiosity → Anxiety → Understanding → Confidence → Relief**

| Step | Screen | User Action | System Response | Emotional State | Trust Element |
| :--- | :----- | :---------- | :-------------- | :-------------- | :------------ |
| 1 | **Landing Page** | User lands on LegacyGuard.com | Engaging hero section: "Your Digital Legacy, Secured." Clear value proposition, testimonials, security badges (e.g., "Audited by CertiK"). | Curiosity, Initial Interest | Social proof, Authority (security audits), Clear value proposition. |
| 2 | **Landing Page** | Clicks "Get Started" or "Sign Up" | Navigates to Sign-Up screen. | Motivation, Anticipation | Clear call to action, seamless transition. |
| 3 | **Sign Up** | Enters email and password, or chooses social login (Google/Apple). | Email verification sent, or social account linked. | Mild Anxiety (data entry), Hope | Standard security practices (email verification), convenience of social login. |
| 4 | **Email Verification** | Clicks link in verification email. | Confirms email, redirects to "Welcome" screen. | Relief, Validation | Standard security protocol, immediate feedback. |
| 5 | **Welcome & Security Intro** | Reads "Welcome to LegacyGuard!" and sees prompt for security education. | Short, engaging text: "Understand how we protect your legacy." Button: "Learn More (30s)." | Understanding, Mild Concern | Proactive security education, transparency. |
| 6 | **Security Explainer** | Watches animated 30s explainer video on encryption, key splitting, and non-custodial nature. | Video plays, visually demonstrating complex concepts simply. | Understanding, Growing Confidence | Visual learning, simplification of complex security, emphasis on non-custodial. |
| 7 | **How LegacyGuard Protects You** | Reads summary of security features: "Your assets are always yours," "Multi-party security," "Audited smart contracts." | Clear bullet points, reassuring tone. | Confidence, Reassurance | Explicit statement of security principles, transparency about audits. |
| 8 | **Wallet Connection** | Clicks "Connect Wallet." Chooses WalletConnect or manual input. | WalletConnect QR code displayed, or input fields for manual connection. | Mild Anxiety (technical step), Determination | Familiar interface (WalletConnect), clear instructions. |
| 9 | **Wallet Connection** | Scans QR code with mobile wallet or inputs details. Confirms connection in wallet. | LegacyGuard confirms wallet address connection. | Relief, Empowerment | Real-time feedback, user control over their assets. |
| 10 | **Vault Creation** | Enters a name for their vault (e.g., "My Crypto Legacy"), selects desired blockchain networks (e.g., Ethereum, Polygon). | Displays selected chains, confirms vault name. | Control, Ownership | User-defined naming, explicit choice of chains. |
| 11 | **Key Splitting Ceremony** | Clicks "Create Vault." Sees an animation of a digital key splitting into three distinct shards. | Visual metaphor: "Your key is now split into 3 secure shards." Explains each shard's purpose (Owner, Heir, LegacyGuard). | Fascination, Understanding | Powerful visual metaphor, clear explanation of security mechanism. |
| 12 | **First Heir Invitation** | Prompted to invite their first heir. Enters heir's name and email. | Confirmation message: "Invitation sent to [Heir's Name]." | Responsibility, Hope | Guided first step, immediate feedback on invitation. |
| 13 | **Plan Activation** | Clicks "Activate Plan." | System confirms plan is active, ready for asset definition and trigger setup. | Relief, Accomplishment | Clear confirmation, sense of progress. |
| 14 | **Dashboard Arrival** | Lands on personalized dashboard showing vault status, pending actions, and quick links. | Dashboard displays "Vault: My Crypto Legacy (Active)," "1 Heir Invited," "Next Step: Define Assets." | Confidence, Direction | Clear overview, guided next steps, sense of control. |
| 15 | **Dashboard Arrival** | Explores dashboard, sees support options. | Access to FAQs, live chat, and tutorials. | Relief, Security | Accessible support, reassurance that help is available. |

## FLOW 2 — Inheritance Plan Creation (10 steps)
**Emotional arc: Determination → Thoughtfulness → Control → Assurance**

| Step | Screen | User Action | System Response | Emotional State | Trust Element |
| :--- | :----- | :---------- | :-------------- | :-------------- | :------------ |
| 1 | **Dashboard** | Clicks "Create New Plan" or "Define Assets" from dashboard. | Navigates to "Plan Setup: Select Vault" screen. | Determination, Focus | Clear navigation, logical next step. |
| 2 | **Select Vault** | Selects the vault created during onboarding (e.g., "My Crypto Legacy"). | Displays selected vault, proceeds to "Add Heirs" screen. | Control, Clarity | Explicit choice, confirmation of selection. |
| 3 | **Add Heirs** | Enters name, email, and relationship for multiple heirs. Clicks "Add Heir." | Displays list of added heirs. Option to add more or proceed. | Thoughtfulness, Responsibility | Clear input fields, immediate feedback on added heirs. |
| 4 | **Define Asset Scope** | Chooses between "All assets in connected wallet" or "Specific assets (manual selection)." | If "Specific assets," prompts for token/NFT selection. If "All assets," confirms scope. | Control, Precision | Granular control over asset inclusion, clear options. |
| 5 | **Choose Trigger Type** | Selects "Inactivity Period" as the trigger for inheritance. | Explains "Dead Man's Switch" concept briefly. Proceeds to "Set Check-in Frequency." | Understanding, Foresight | Clear explanation of trigger mechanism, user choice. |
| 6 | **Set Check-in Frequency** | Uses a slider or input to set check-in frequency (e.g., every 30, 60, 90 days). | Displays chosen frequency. | Control, Customization | Intuitive control (slider), explicit choice. |
| 7 | **Configure Verification** | Chooses verification methods for heirs (e.g., Government ID check, Security Questions, Trusted Contacts confirmation). | Explains each method's purpose and security benefits. | Security, Responsibility | Transparency about verification, user choice for security layers. |
| 8 | **Review Plan** | Reviews all configured details: Vault, Heirs, Assets, Trigger, Verification. | Presents a comprehensive summary of the plan. "Looks good?" button. | Assurance, Scrutiny | Comprehensive summary, opportunity for correction. |
| 9 | **Confirm with 2FA** | Clicks "Confirm Plan." Prompted for 2FA code (e.g., Authenticator app). | Verifies 2FA code. | Security, Finality | Strong security measure for critical action. |
| 10 | **Plan Activated Confirmation** | Sees a confirmation screen: "Your Inheritance Plan is Active!" with a unique plan ID. | Displays next steps (e.g., "Share instructions with heirs"). | Relief, Accomplishment | Clear confirmation, unique identifier, guidance for next steps. |

## FLOW 3 — Trigger Setup (8 steps)
**Emotional arc: Deliberation → Responsibility → Foresight → Peace of Mind**

| Step | Screen | User Action | System Response | Emotional State | Trust Element |
| :--- | :----- | :---------- | :-------------- | :-------------- | :------------ |
| 1 | **Dashboard / Plan Details** | Clicks "Configure Trigger" for an existing plan. | Navigates to "Trigger Setup: Inactivity Threshold." | Deliberation, Focus | Clear entry point, context of existing plan. |
| 2 | **Select Inactivity Threshold** | Uses a slider to set the inactivity threshold (e.g., 30, 90, 180, 365 days). | Displays selected duration and a brief explanation of its implications. | Responsibility, Foresight | Intuitive slider, clear explanation of consequences. |
| 3 | **Choose Check-in Methods** | Selects preferred check-in methods (e.g., Email, Push Notification, SMS, Hardware Key confirmation). | Displays selected methods, explains reliability of each. | Control, Customization | Multiple options, transparency about method reliability. |
| 4 | **Set Fallback Verification** | Configures a fallback method if primary check-in fails (e.g., biometric scan, security questions). | Explains the importance of fallback for account security. | Security, Preparedness | Redundancy for security, clear rationale. |
| 5 | **Configure Heir Notification Timing** | Sets the delay for heir notification *after* the inactivity threshold is met (e.g., immediate, 24h, 72h). | Explains the purpose of a delay (e.g., "allows time for owner to intervene"). | Thoughtfulness, Control | Granular control over sensitive timing, clear explanation. |
| 6 | **Preview: What Happens If You Don't Check In** | Clicks "Preview Sequence." | Displays a step-by-step visual timeline: "Inactivity detected → Check-in attempts → Fallback verification → Heir notification delay → Heir claim process." | Clarity, Reassurance | Visual timeline, transparent process, removes ambiguity. |
| 7 | **Activate Trigger** | Clicks "Activate Trigger" after reviewing the preview. | System confirms trigger activation, prompts for 2FA. | Finality, Security | Explicit activation, strong security for critical action. |
| 8 | **Trigger Activated Confirmation** | Sees confirmation: "Trigger for [Plan Name] is now Active." | Displays current status and next check-in date. | Peace of Mind, Assurance | Clear confirmation, ongoing status visibility. |

## FLOW 4 — Heir Claim Process (12 steps)
**Emotional arc: Grief/Shock → Uncertainty → Hope → Relief → Gratitude**

| Step | Screen | User Action | System Response | Emotional State | Trust Element |
| :--- | :----- | :---------- | :-------------- | :-------------- | :------------ |
| 1 | **Heir Notification (Email/App)** | Heir receives a sensitive notification (email and/or app push) about the owner's passing and the inheritance process. | Message is empathetic, respectful, and clear: "We regret to inform you..." Links to LegacyGuard claim portal. | Grief, Shock, Confusion | Empathetic language, clear source (LegacyGuard), direct link to next steps. |
| 2 | **Grief-Aware Landing Page** | Clicks link, lands on a dedicated, somber-toned landing page. | Page offers condolences, explains the purpose of the platform, and guides them gently to start the claim. "Take your time." | Grief, Uncertainty, Mild Hope | Empathetic design, respectful tone, non-pressuring language, clear purpose. |
| 3 | **Start Claim Process** | Clicks "Begin Claim." | Navigates to Identity Verification screen. | Determination (mixed with grief), Anxiety | Clear call to action, gentle progression. |
| 4 | **Identity Verification** | Uploads government ID (e.g., passport, driver's license) and performs a liveness check (e.g., selfie scan). | System processes ID, provides real-time feedback on verification status. | Anxiety, Frustration (if issues), Hope | Standard security practice, clear instructions, real-time feedback. |
| 5 | **Security Questions** | Answers pre-configured security questions set by the owner. | System verifies answers against owner's records. | Anxiety, Focus | Personalized security, reinforces owner's intent. |
| 6 | **Waiting Period** | Sees a screen indicating a mandatory waiting period (e.g., "48-hour safety window"). | Explains the purpose: "Allows the owner a final chance to cancel if this is a false alarm." Displays countdown. | Impatience, Mild Anxiety, Understanding | Transparency about the delay, clear safety rationale, countdown timer. |
| 7 | **Multi-Party Confirmation (if configured)** | If applicable, sees prompt to await confirmation from other designated trusted contacts. | Displays status of confirmations received/pending. | Impatience, Reliance | Distributed trust, adds layer of security, transparency of status. |
| 8 | **Key Shard Assembly** | Once all conditions met, sees an animation of the three key shards (Owner, Heir, LegacyGuard) visually coming together to form a complete key. | "Your key is now assembled." Explains what this means: access granted. | Relief, Empowerment, Fascination | Powerful visual metaphor, clear explanation of success. |
| 9 | **Recovery Instructions Revealed** | Displays clear, step-by-step instructions on how to access the inherited assets (e.g., "Connect your wallet," "Sign transaction"). | Provides necessary details (e.g., contract addresses, specific wallet actions). | Relief, Clarity | Explicit, actionable instructions, removes guesswork. |
| 10 | **Asset Access Guidance** | Offers direct links or guidance to common crypto wallets/exchanges for asset management. | "Need help moving your assets?" Links to tutorials. | Empowerment, Practicality | Proactive support for next steps, reduces friction. |
| 11 | **Support Resources** | Provides links to grief counseling services, financial advisors specializing in crypto, and LegacyGuard's support team. | "We're here to help you through this." | Gratitude, Support | Holistic support beyond just crypto, empathetic outreach. |
| 12 | **Completion** | Sees a "Claim Complete" screen. | Offers option to close the session or review transaction history. | Relief, Closure | Clear end point, sense of accomplishment. |

## FLOW 5 — Emergency Recovery (8 steps)
**Emotional arc: Panic/Urgency → Relief → Control → Security**

| Step | Screen | User Action | System Response | Emotional State | Trust Element |
| :--- | :----- | :---------- | :-------------- | :-------------- | :------------ |
| 1 | **Login / Recovery Portal** | Owner logs in or accesses a dedicated "Emergency Recovery" link (e.g., after a false trigger). | Navigates to "Prove Liveness" screen. | Urgency, Panic (if false trigger) | Clear, direct path to recovery, immediate action. |
| 2 | **Prove Liveness** | Performs biometric scan (e.g., face ID, fingerprint) and enters 2FA code. | System verifies biometrics and 2FA. | Anxiety, Focus | Strong, multi-factor authentication for critical action. |
| 3 | **Cancel Pending Claim** | If a claim is pending, sees a clear option: "A claim is pending for [Heir's Name]. Do you wish to cancel it?" Clicks "Yes, Cancel Claim." | System immediately cancels the pending claim, sends notification to heir (if configured). | Relief, Control | Immediate action, clear consequence, prevents false inheritance. |
| 4 | **Update Plan (Optional)** | If no claim was pending, or after cancelling, offered option to review/update their plan. Clicks "Update Plan." | Navigates to Plan Details screen. | Control, Foresight | User control to modify plan, proactive suggestion. |
| 5 | **Re-verify Identity** | If significant changes are made or for security, prompted for a full identity re-verification (e.g., government ID upload). | System processes ID, confirms identity. | Mild Frustration, Security | Enhanced security for critical plan changes, transparency. |
| 6 | **Confirmation of Action** | Sees confirmation: "Claim for [Heir's Name] has been cancelled" or "Your plan has been updated." | Displays current plan status. | Relief, Assurance | Clear, immediate feedback on the action taken. |
| 7 | **Security Review** | Prompted to review security settings (e.g., check-in methods, trusted contacts). | Suggests strengthening security if needed. | Security, Responsibility | Proactive security recommendations, user empowerment. |
| 8 | **Dashboard Return** | Returns to the main dashboard. | Dashboard reflects updated status (e.g., "No pending claims"). | Control, Peace of Mind | Clear status update, return to familiar interface. |

## Journey Summary

| Flow | Steps | Est. Duration | Complexity | Emotional Arc | Key Risk |
| :--- | :---- | :------------ | :--------- | :------------ | :------- |
| **FLOW 1 — New User Onboarding** | 15 | 10-15 min | Medium | Curiosity → Anxiety → Understanding → Confidence → Relief | User drops off due to perceived complexity or security concerns. |
| **FLOW 2 — Inheritance Plan Creation** | 10 | 15-20 min | Medium | Determination → Thoughtfulness → Control → Assurance | Incorrect plan setup (e.g., wrong heirs, asset scope), or user feels overwhelmed. |
| **FLOW 3 — Trigger Setup** | 8 | 5-10 min | Low-Medium | Deliberation → Responsibility → Foresight → Peace of Mind | Trigger set too short/long, or check-in methods are unreliable, leading to false triggers or missed inheritance. |
| **FLOW 4 — Heir Claim Process** | 12 | 20-30 min | High | Grief/Shock → Uncertainty → Hope → Relief → Gratitude | Heir unable to claim due to verification issues, or fraudulent claim attempt. High emotional distress for the user. |
| **FLOW 5 — Emergency Recovery** | 8 | 5-10 min | Medium | Panic/Urgency → Relief → Control → Security | Owner unable to recover control due to forgotten credentials or failed liveness proof, or false recovery attempt. |
