# MVP UI Scope & Implementation Guide
*LegacyGuard Product Design — Generated 2026-03-17*

---

LegacyGuard's MVP UI is meticulously scoped to deliver an Apple-level experience for a critical, sensitive domain: crypto inheritance. Our focus is on absolute clarity, unwavering security, and a user experience that instills trust and control, avoiding the common pitfalls of DeFi.

This document outlines the precise UI scope, component mapping for a React/Next.js implementation using `shadcn/ui`, and the necessary design deliverables to ensure a smooth, high-quality build.

---

## LegacyGuard MVP UI Scope: Detailed Definition

### Core Principles Guiding MVP UI
*   **Emotional Intelligence:** Acknowledging the sensitive nature of inheritance, language is empathetic, clear, and reassuring.
*   **Progressive Security:** Complex security concepts (e.g., Shamir Secret Sharing) are introduced gradually, with visual aids and simplified explanations.
*   **Trust-First Design:** Security status, non-custodial nature, and audit information are always prominent.
*   **Non-Custodial Transparency:** Users always understand that LegacyGuard never holds their keys or assets.
*   **Engagement through Check-in:** The "dead man's switch" is framed as a proactive measure for peace of mind, not a burden.

---

### MVP UI: What's IN (Screens & Key Elements)

#### Screen 1: Landing / Onboarding
*   **Purpose:** Introduce LegacyGuard, build trust, and guide users to sign up/sign in.
*   **Key Elements:**
    *   Compelling value proposition (3 steps visual, e.g., "Set up in minutes," "Secure with advanced crypto tech," "Peace of mind for your family").
    *   Prominent 'Protect Your Crypto Legacy' Call to Action (CTA).
    *   Trust signals: Security badges (e.g., "Audited by [Firm]"), partnership logos, clear privacy statement link.
    *   Sign up / Sign in options (email/password, social login if supported by auth provider).

#### Screen 2: Onboarding Wizard (4 steps)
*   **Purpose:** Guide new users through the essential initial setup in a clear, step-by-step manner.
*   **Key Elements:**
    *   **Step 1: Welcome + Security Education:** Brief, engaging intro to LegacyGuard's non-custodial security model (e.g., "Your keys, your control. We never touch your crypto."). Visuals explaining 2-of-3 Shamir Secret Sharing at a high level.
    *   **Step 2: Connect Wallet:** Primary method via WalletConnect v2. Manual address input as a fallback. Clear instructions and security warnings.
    *   **Step 3: Create Vault:** User names their vault (e.g., "My Main Vault," "Family Legacy"). Selects initial blockchain networks to cover (e.g., Ethereum, Polygon).
    *   **Step 4: Invite First Heir:** Input for heir's email address. Clear explanation of what happens next (heir receives invitation).

#### Screen 3: Dashboard (simplified)
*   **Purpose:** Provide a quick overview of the user's legacy plan status and key actions.
*   **Key Elements:**
    *   **Security Score/Status:** Prominent indicator (e.g., "Excellent," "Good," "Needs Attention") reflecting vault setup completeness, heir verification, etc.
    *   **Active Plan Summary:** Card displaying the vault name, number of heirs, and trigger status.
    *   **Next Check-in Countdown:** Clear display of when the next check-in is due.
    *   **Quick Actions:** Prominent 'Check In' button. 'Edit Plan' (links to Vault/Heir/Trigger config), 'View Activity' (links to activity log).

#### Screen 4: Vault Setup
*   **Purpose:** Allow users to manage their vault's core configuration.
*   **Key Elements:**
    *   **Vault Name:** Editable field.
    *   **Connected Wallets:** List of connected wallet addresses, with option to add/remove (with confirmation).
    *   **Chains Covered:** List of selected blockchain networks, with option to add/remove.
    *   **Encryption Status Indicator:** Visual confirmation that the vault is encrypted client-side (e.g., "Vault Encrypted: AES-256-GCM").
    *   **Key Shard Distribution Visual:** Simplified visual representation of the 2-of-3 SSS (e.g., "You hold 1 shard, LegacyGuard holds 1 shard, Your Heir holds 1 shard").

#### Screen 5: Heir Configuration
*   **Purpose:** Manage the list of designated heirs.
*   **Key Elements:**
    *   **Add Heir Flow:** Modal/form for Name, Email, Relationship (e.g., "Spouse," "Child," "Friend").
    *   **Heir List:** Table/list displaying each heir with their status (e.g., "Invited," "Verified," "Active").
    *   **Actions:** 'Remove Heir' and 'Edit Heir' options (with clear confirmation flows for removal).

#### Screen 6: Trigger Configuration
*   **Purpose:** Define the "dead man's switch" logic.
*   **Key Elements:**
    *   **Inactivity Period Selector:** Slider or input for setting the number of days (e.g., 30, 60, 90, 180, 365 days) after which the trigger activates.
    *   **Check-in Method Selection:** Radio buttons/checkboxes for preferred check-in notification methods (e.g., Email, Push Notification).
    *   **Dynamic Preview:** A clear, human-readable sentence summarizing the configured trigger (e.g., "If you don't check in for 90 days, your heirs will be notified via email.").

#### Screen 7: Check-in Screen
*   **Purpose:** The primary interaction point for users to confirm their presence and reset the trigger.
*   **Key Elements:**
    *   **Prominent 'I'm Here' Button:** Large, clear, and easy to tap/click.
    *   **Last Check-in Timestamp:** Display of the most recent check-in date/time.
    *   **Next Check-in Due Date:** Clear indication of when the next check-in is expected.
    *   **Activity Log:** Simple list of past check-ins.

#### Screen 8: Notifications
*   **Purpose:** Centralized view of important alerts and updates.
*   **Key Elements:**
    *   **Check-in Reminders:** Notifications about upcoming or overdue check-ins.
    *   **Security Alerts:** Warnings about suspicious activity, failed login attempts, etc.
    *   **Heir Activity Updates:** Notifications when an heir accepts an invitation, verifies identity (if applicable in future), or initiates a claim (future).

#### Screen 9: Settings
*   **Purpose:** Manage user account preferences.
*   **Key Elements:**
    *   **Profile:** Basic user information (name, email).
    *   **Security:** 2-Factor Authentication (2FA) setup/management. Password change.
    *   **Notifications Preferences:** Toggles for email, push notification types.

### MVP UI: What's EXPLICITLY OUT
*   Multi-vault management (v2)
*   Advanced trigger logic (multi-condition, oracle-based)
*   Partner dashboard (v2)
*   Developer portal (v2)
*   Multi-chain automatic detection
*   Video verification for heirs
*   Mobile native app (web-first)
*   Social recovery module
*   Token-based governance
*   On-chain smart contract triggers
*   **Heir Claim Process UI (for MVP)** - While the *system* handles it, the UI for heirs to initiate and complete a claim is out of the *MVP UI scope* for the primary user app. This will be a separate, highly secure, and guided flow developed post-MVP.

---

### Tech Stack for MVP UI
*   **Framework:** Next.js 14+ (App Router)
*   **Styling:** Tailwind CSS + shadcn/ui components
*   **State Management:** Zustand
*   **Authentication:** Supabase Auth or Clerk
*   **Wallet Integration:** WalletConnect v2 + wagmi
*   **Animations:** Framer Motion
*   **Icons:** Lucide
*   **Deployment:** Vercel
